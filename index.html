環境配置：

1.安裝Node.js

筆者在安裝來源為Node.js官網，嘗試好幾次仍然跳出錯誤。

依照Hexo官網的指示，輸入下列指令即可

至於為什麼用nvm，可以查看這篇用nvm install node 安裝Node.js

2.安裝Hexo Git

npm install hexo-deployer-git --save

如果此時你發現報錯，而且提示為gyp: No Xcode or CLT version detected!(未檢測到Xcode或CLT版本)

就筆者的狀況來說有二：

首先是檢查MacOS是否更新到最新版本，但更新完後輸入上面的指令依然報錯。

第二是我原先沒有安裝Xcode，但安裝後又顯示上述報錯，這不是很奇怪？搜尋了一下發現有蠻多人面臨相同問題，大多數的解決方式為sudo xcode-select --install，這篇提到可能是Mac升級後，缺了xcode 的CLI 工具，但我輸入這個指令還是有報錯的提示。
:::danger
xcode-select: error: command line tools are already installed, use “Software Update” to install updates
:::

依據這篇執行sudo rm -rf /Library/Developer/CommandLineTools，移除後xcode-select --install重新執行成功。

報錯解決後，重新執行npm install hexo-deployer-git --save

3.安裝Hexo

  • npm install hexo-cli -g 或 sudo npm install -g hexo-cli
  • hexo init [部落格名稱]
  • cd [部落格名稱] # 移至剛創建的資料夾
  • npm install # 安裝相關套件

4.將Hexo配置到Github上

登入Github新增一個Repositories，名稱為Github帳號名.github.io

回到專案文件夾，點開全域配置文件_config.yml，將文件最下方原先空白的deploy，修改如下：

冒號後一定要空格

不確定repository的url？到Github

1.hexo g產生靜態檔案

配置完成後的文章更新方式，參考在Hexo博客中发布文章

2.hexo new 文章名稱

路徑：專案文件夾/source/_posts

打開新增的page， Markdown語法編輯內文

範例：

3.hexo s =>啟動server，本地測試

hexo server
查看文章與內容是否成功建立，預設是 http://localhost:4000/

[報錯]開其他的porthexo server -p 5000即可

4.hexo d =>產生完成即部署網站

hexo deploy
進到 https://github名稱.github.io/

如果此時你發現報錯，可能是配置文件的空格或註解所導致

  • hexo deploy不能用？
  • 使用hexo时遇到的问题

使用主題Next

參考Next官方網站

  • 進到文件 cd [部落格名稱]
  • clone主題 git clone https://github.com/iissnan/hexo-theme-next themes/next

    路徑：文件夾/_config.yml

  • 在全域配置文件將預設主題改為Next

主題可選擇想要的外觀Scheme，並將註解拿掉(擇一不能重複)

路徑：專案文件夾/themes/next/_config.yml

重新啟動Server :hexo s
